# ZinPass 密码管理器

## 📇 一、简介
ZinPass 是一款自用的密码管理器，分为三部分：
- 客户端：zinpass.exe 负责前端交互，使用 Qt6 框架
- 服务端：zinpasssvc.exe 负责后端数据处理，使用 gRPC 与 前端以及命令行工具通信，使用 SQLite3 存储数据，使用 OpenSSL 加密数据（目前版本只加密密码字段，后续版本可能加密整个文件）。
- 服务端命令行交互工具：zinpassctl.exe 负责监控和控制服务端后台进程。服务端没有控制台和图形化界面，监控和控制需要此工具。
这个程序只实现了少数几个最基本的功能，还有很多功能未实现，目前没啥精力更新，且每次更新大概率都是破坏性更新，不保证兼容上一版本。

## 🛠️ 二、构建
使用 CMake + MinGW-w64 构建，构建前先运行 protoc.py 脚本生成 gRPC 的桩代码，然后再构建。

## ⚙️ 三、使用
先双击 zinpasssvc.exe 运行（不会显示任何控制台或图形界面，后台运行），然后启动 zinpass.exe 进行增删查改。
关闭 zinpasssvc 服务需要使用 zinpassctl.exe 工具，在该工具的交互命令中，输入 `stop` 命令控制 zinpasssvc 停止服务。
需要注意的是，会话 30min 自动过期，会话过期时查询不到任何数据，客户端也不会提示会话过期，请自行退出重新登录。这一问题将在后续版本中解决。
密码数据不上传任何其他服务器，一切都运行在本地主机，要备份数据直接备份 zinpass.db 这个文件就行，但是因为在发布正式版本之前，都不保证兼容前一版本，备份 zinpass.db 文件前请同时记录 zinpass.db 是配套哪个版本的软件。
